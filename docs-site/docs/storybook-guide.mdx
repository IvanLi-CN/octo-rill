# Storybook 导览

在 OctoRill 里，Storybook 用来回答三个问题：页面有哪些稳定状态、管理员界面如何在复杂数据下表现、基础组件在业务语境里长什么样。

## 什么时候打开 Storybook

- 你在改 UI，需要先确认现有页面状态。
- 你在补 stories、docs 或交互覆盖。
- 你想验证某个文案、空态、加载态或抽屉状态，而不想先跑完整业务数据。

## 与公开文档的分工

- 公开文档解释产品、启动方式和配置。
- Storybook 展示页面状态、组件组合和视觉边界。

当你需要“解释为什么有这个页面”，先回到 [产品说明](/product)；当你需要“确认这个状态长什么样”，直接留在 Storybook。
