# 规格（Spec）总览

本目录维护 OctoRill 的 topic-level spec catalog。若你要找的是项目入口、系统概览或内部维护文档，请先回到 [`../README.md`](../README.md)。

- `SPEC.md`：长期有效的主题规范、契约、范围与验收口径。
- `IMPLEMENTATION.md`：当前实现覆盖、验证结果、交付状态与剩余缺口。
- `HISTORY.md`：关键演进原因、替代关系与历史事实。

## 目录约定

- 每个主题使用一个稳定目录：`docs/specs/<id>-<title>/`
- `<id>` 保持既有 spec id，不因 hygiene 迁移重新分配。
- companion docs 固定为 `SPEC.md`、`IMPLEMENTATION.md`、`HISTORY.md`。

## Lifecycle

- `active`：当前主题仍是有效规范，后续实现或 review 应继续对齐该目录。
- `superseded(#<id>)`：该主题已被后继 spec 取代，保留历史参考。
- `archived`：仅保留历史事实，不再作为当前规范入口。
